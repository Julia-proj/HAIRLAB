## Архитектура AI-слоя

### Файловая структура
```
lib/ai/
├── knowledge-base.ts    # Структурированные кейсы Елены
├── prompts/
│   ├── system.ts        # Базовый system prompt (тон, роль, ограничения)
│   ├── classification.ts # Промпт для классификации по фото
│   ├── recommendations.ts # Промпт для рекомендаций
│   └── quiz-report.ts   # Промпт для AI-отчёта после квиза
├── diagnosis.ts         # Основной pipeline: фото → результат
├── streaming.ts         # SSE streaming для UI
├── guardrails.ts        # Валидация, rate limiting, safety
└── types.ts             # DiagnosisResult, Classification и т.д.
```

### Multi-step pipeline
```
Шаг 1: Классификация (vision)
  Input: 1-3 фото + результат квиза
  Model: Claude Opus 4.6 с vision
  Output: { damageLevel, hairType, porosity, signs[], zones[] }

Шаг 2: Рекомендации (text)
  Input: результат классификации + knowledge base (few-shot)
  Model: Claude Opus 4.6
  Output: персональный текст в стиле Елены

Шаг 3: Форматирование
  Input: рекомендации + ссылки на уроки курса
  Output: структурированный отчёт для UI
```

### Knowledge Base (few-shot)
Каждый кейс Елены структурируется так:

```typescript
interface ExpertCase {
  id: string
  photoDescription: string  // "Осветлённый блонд, 8 уровень, видна пористость на концах"
  classification: {
    damageLevel: 1 | 2 | 3 | 4 | 5
    hairType: 'natural' | 'colored' | 'bleached' | 'heavily_damaged'
    porosity: 'low' | 'medium' | 'high'
    signs: string[]  // ["сухость", "ломкость", "отсутствие эластичности"]
  }
  recommendation: string   // Текст Елены: что делать, какие составы, чего избегать
  commonMistakes: string[] // Что обычно советуют неправильно для этого случая
}
```

В system prompt передаём 10-15 кейсов как примеры. Claude видит паттерн и отвечает в стиле Елены.

## Правила промптов

### System prompt (базовый тон)
```
Ты помощник мастера по восстановлению волос Елены (студия Keratin Madrid, Мадрид).
Твоя задача: оценить состояние волос по фото и дать рекомендации.

Ты НЕ заменяешь мастера. Ты даёшь предварительную оценку.
В конце ВСЕГДА говори: "Для точной диагностики рекомендую консультацию с мастером."

Тон: тёплый, уверенный, конкретный. Как подруга-эксперт.
Обращение: на "ты".
Без длинного тире.
Без: "важно понимать", "в современном мире", "уникальная методика".

Ты обучена на реальных кейсах Елены (ниже). Отвечай в её стиле и логике.
```

### Промпт классификации (vision)
```
Посмотри на фото волос.

Определи:
1. Степень повреждения (1-5): 1 = минимальное, 5 = сильное
2. Тип: натуральные / окрашенные / осветлённые / сильно повреждённые
3. Пористость: низкая / средняя / высокая
4. Видимые признаки (перечисли конкретно)
5. Проблемные зоны: корни / длина / концы

Ответь строго в JSON:
{
  "damageLevel": число,
  "hairType": "тип",
  "porosity": "уровень",
  "signs": ["признак1", "признак2"],
  "zones": ["зона1", "зона2"],
  "confidence": "low" | "medium" | "high"
}

Если фото нечёткое или не получается определить, поставь confidence: "low".
```

### Промпт рекомендаций
```
На основе диагностики:
[вставляем JSON из шага 1]

И на основе результатов квиза:
[вставляем ответы квиза]

Дай персональные рекомендации. Формат:

1. Что происходит с волосами (2-3 предложения, простым языком)
2. Что делать в первую очередь (1 конкретное действие)
3. Какой урок курса посмотреть первым (номер + название)
4. Чего точно НЕ делать (1-2 пункта)
5. Прогноз: через сколько будет заметен результат

Пиши как Елена: конкретно, тепло, без воды.
Заверши словами: "Для точной диагностики рекомендую консультацию с мастером."

Примеры того, как Елена отвечает:
[вставляем 5-10 кейсов из knowledge base]
```

## Guardrails

### Что проверяем
1. **Фото валидно:** есть волосы на фото (не кот, не еда)
2. **Размер:** не больше 10MB на фото
3. **Количество:** 1-3 фото за сессию
4. **Rate limit:** 3 диагностики в день на пользователя
5. **Confidence:** если confidence = "low", говорим "фото недостаточно чёткое, попробуй переснять"

### Чего AI НИКОГДА не делает
- Не ставит медицинских диагнозов ("у вас алопеция")
- Не рекомендует лекарства
- Не говорит "ваши волосы безнадёжны"
- Не критикует конкретные бренды ("Kérastase вам не подходит")
- Не обещает конкретных сроков ("через 2 недели будет идеально")
- Всегда добавляет дисклеймер про консультацию с мастером

## Streaming

AI-ответ должен появляться в UI постепенно (по токенам), не "подождите 15 секунд".

Техническая реализация:
- POST /api/ai/diagnose → возвращает ReadableStream
- Frontend: EventSource или fetch + ReadableStream reader
- Каждый chunk рендерится по мере получения
- Показываем animated gradient mesh фон пока генерируется

## Работа с датасетом Елены

Когда датасет придёт (фото + аудио + FAQ):
1. Фото: описать каждое текстом (для few-shot, Claude не хранит фото между сессиями)
2. Аудио: транскрибировать (Whisper), вытащить ключевые паттерны
3. FAQ: структурировать в формат question-answer
4. Собрать 20-30 ExpertCase объектов
5. Выбрать 10-15 лучших для system prompt
6. Остальные использовать для тестирования
