# 03 — UX / User Flow

## Эмоциональный и логический путь пользователя

---

## Философия UX

Этот продукт не показывает информацию. Он **ведёт человека**.

Каждый экран, каждый шаг, каждая кнопка отвечает на три вопроса:
1. Где я сейчас?
2. Зачем мне это?
3. Что делать дальше?

Если на любом этапе человек чувствует "а зачем я здесь?" — мы его потеряли.

---

## Полная карта пути

```
[Reels/TikTok]
     ↓ узнавание + доверие
[Landing: Guided Flow]
     ↓ боль → понимание → эксперт → квиз → ценность → оффер
[Quiz + Бесплатный урок]
     ↓ "если бесплатное так полезно..."
[Оплата]
     ↓ Stripe Checkout
[Onboarding]
     ↓ welcome + первый шаг
[Dashboard: Обучение]
     ↓ уроки → прогресс → бонусы
[Удержание]
     ↓ возвращение, рекомендации, upsell
```

## Этап 1: Привлечение (Reels / TikTok)

### Где происходит
Instagram Reels, TikTok, Stories

### Эмоция пользователя
"О, это же про мои волосы!" → "Она реально разбирается" → "Хочу узнать больше"

### Задача
Три вещи за 30–60 секунд:
1. Узнавание ("это про меня")
2. Экспертность ("она знает, о чём говорит")
3. Действие ("напишу ДИАГНОСТИКА / перейду по ссылке")

### Точка доверия
Реальные примеры, реальные волосы, реальные объяснения. Не постановка.

### Переход
Ссылка в био → сайт. Или комментарий "ДИАГНОСТИКА" → директ с ссылкой.

## Этап 2: Guided Landing Flow

### Принцип
Не лендинг-простыня. Последовательность экранов. Один экран = одна мысль = один следующий шаг.

### Эмоциональный маршрут

```
Экран 1: "Это про меня"          → узнавание
Экран 2: "Теперь понимаю почему"  → осознание
Экран 3: "Вот почему ничего не помогало" → инсайт
Экран 4: "Ей можно доверять"      → доверие
Экран 5: "Хочу проверить свои"    → вовлечение (квиз)
Экран 6: "Вот что с моими волосами" → персонализация
Экран 7: "Это реально полезно"    → ценность (бесплатный урок)
Экран 8: "Хочу всё"              → желание
Экран 9: "Это стоит того"        → уверенность
Экран 10: "Покупаю"              → действие
```

Каждый экран должен **заслужить** переход на следующий. Не вынудить — заслужить.

### UX-правила flow

- Прогресс-бар сверху (тонкий, ненавязчивый)
- Кнопка "Назад" всегда видна
- Кнопка "Перейти к курсу" (skip для готовых)
- Плавные fade-переходы между экранами
- На мобильном — touch-friendly, можно свайпнуть
- Один CTA на экран — всегда внизу, всегда заметный
- Текст читаемый без зума (16px+)

## Этап 3: Quiz (Диагностика)

### Зачем квиз

Квиз — это не маркетинговый трюк. Это реальная ценность:
1. Вовлекает — человек отвечает про себя, а не читает про других
2. Персонализирует — результат про конкретные волосы
3. Обучает — через вопросы человек начинает понимать свою проблему
4. Готовит к покупке — "если диагностика такая полезная, что же в курсе?"

### Эмоция пользователя
"Интересно" → "Хм, действительно, у меня так" → "Хочу увидеть результат"

### UX квиза
- Один вопрос на экран
- Карточки-кнопки для ответов (не dropdown, не чекбоксы)
- Прогресс-бар
- 4–5 вопросов максимум
- Анимация при выборе
- Не больше 30 секунд на весь квиз

---

## Этап 4: Результат + Бесплатный урок

### Результат квиза

Не просто "у вас повреждение 3 из 5".
А полноценный мини-отчёт:

- Визуальная шкала повреждения
- 2–3 предложения: что это значит простыми словами
- Что делать дальше

### Бесплатный урок

**Критически важно:** бесплатный урок должен быть **реально полезным**. Не тизер. Не обрезанное видео. Конкретная информация, которую можно применить сразу.

Содержание: 5 признаков повреждённых волос + тест на эластичность (из контента Елены).
Длительность: 2–3 минуты.

### Эмоция пользователя
"Если бесплатный кусок настолько полезный, в полном курсе должно быть что-то серьёзное."

### Точка доверия
Бесплатный урок — это главная точка доверия. Здесь человек впервые получает реальную пользу. Если эта ступень слабая — конверсия в покупку не сработает.

---

## Этап 5: Оффер и оплата

### Три экрана перед оплатой

**Что внутри:** список уроков, формат, бонусы — чтобы человек видел объём.
**Доказательства:** отзывы, цифры, калькулятор экономии — чтобы снять сомнения.
**Цена и кнопка:** чистый экран, ничего лишнего — чтобы не мешать решению.

### Эмоция пользователя
"Это стоит того" → "Это безопасно" → "Покупаю"

### UX оплаты
- Stripe Checkout (hosted page) — не кастомная форма
- Минимум полей
- Иконки безопасности
- Сразу после оплаты — redirect в dashboard
- Email подтверждение

---

## Этап 6: Onboarding после покупки

### Самый опасный момент

Между оплатой и первым уроком — зона, где теряется больше всего людей. Человек заплатил, но ещё не начал. Если прямо сейчас не дать ему понятный первый шаг — он "отложит на потом" и может не вернуться.

### Что видит пользователь

Один экран:
- "Добро пожаловать!"
- "Курс состоит из 5 уроков. Каждый строится на предыдущем."
- "Начни с первого — он займёт ~15 минут."
- [Кнопка: Начать первый урок]

### Эмоция
"Я всё правильно сделала. Мне понятно, что делать. Начинаю."

---

## Этап 7: Обучение в Dashboard

### Сценарии

**Первый визит:**
Dashboard → Первый урок → Просмотр → "Отметить как пройденный" → Следующий

**Повторный визит:**
Dashboard → "Продолжить [Урок X]" → Продолжает с того места

**Завершение курса:**
Все уроки пройдены → Бонусы → Рекомендации → В будущем: upsell

### Ключевые UX-правила

1. **Continue — первое, что видишь.** Не нужно искать, где остановился.
2. **Прогресс мотивирует.** Визуальная полоска: "3 из 5 — ты на 60%!"
3. **Locked уроки интригуют.** Видно название и описание, но не доступен — мотивация пройти предыдущий.
4. **Summary после урока.** Ключевые выводы + "что сделать сегодня".
5. **Нет тупиков.** Всегда есть следующий шаг.

## Карта точек доверия

Доверие строится постепенно, не за один момент:

| Этап | Точка доверия |
|---|---|
| Reels | Реальные волосы, реальные объяснения, без постановки |
| Landing: экран 4 | Цифры: 6 лет, 5000+ клиентов, Мадрид |
| Quiz result | Персональный результат — "она знает мою проблему" |
| Бесплатный урок | Реальная польза бесплатно — "она не жадничает" |
| Экран доступности | "Этот продукт для таких как я" — снятие барьера "мне это не по карману / не для меня" |
| Отзывы | Скриншоты реальных сообщений клиенток |
| Калькулятор | Конкретные цифры экономии — рациональное подтверждение |
| Stripe | Безопасная оплата, известный бренд |
| Dashboard | Красивый, работающий продукт — "это серьёзно" |

**Ключевой момент:** Экран доступности (Screen 9) — это не про "скидку". Это эмпатия. Пользователь должен почувствовать: "Этот продукт понимает мою ситуацию. Он создан для реальной жизни, где не у всех есть 480€ на салон и не в каждом городе есть хороший мастер."

## Где теряем людей — и как не терять

| Проблема | Решение |
|---|---|
| Flow длинный, бросают | Прогресс-бар + skip link |
| Квиз непонятен | Простые вопросы, визуальные карточки |
| Бесплатный урок не грузится | Быстрый embed, превью-картинка |
| Оплата пугает | Stripe hosted (знакомый), иконки безопасности |
| После покупки непонятно | Мгновенный redirect + welcome screen |
| Вернулся через неделю | "Continue where you left off" первым блоком |
| Забыл пароль | Google OAuth (нет пароля) |
| Мобильный экран тесный | Touch-friendly кнопки, sticky CTA внизу |

---

## Мобильный UX — приоритеты

80%+ трафика будет с телефона. Каждое решение — mobile-first.

- Кнопки: минимум 44px
- Текст: 16px минимум, хороший контраст
- CTA: sticky внизу экрана (не теряется при скролле)
- Видео: адаптивный player, lazy load
- Навигация: bottom nav в dashboard
- Никаких popup'ов
- Свайп между экранами flow
