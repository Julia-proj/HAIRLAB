## Mindset

Этот проект — не "сайт для клиента". Это **первая реализация паттерна**:

> Эксперт + Система + Технология = Масштабируемый digital product

## AI в продукте — философия

### AI = усилитель, не замена

AI не заменяет Елену. AI помогает масштабировать её экспертизу:
- Елена может провести 5 диагностик в день. AI — 500.
- Елена подбирает средства одной клиентке за 20 минут. AI — за 20 секунд.
- Но AI работает на базе знаний Елены. Без её экспертизы AI — просто ещё один ChatGPT.

### Что AI делает хорошо
- Первичная оценка по фото
- Подбор продуктов по параметрам
- Ответы на типовые вопросы
- Персонализация рекомендаций

### Что AI не делает
- Не ставит окончательных диагнозов
- Не заменяет консультацию мастера
- Не принимает решений за пользователя
- Всегда: "Для точной диагностики рекомендуем мастера"

## Направление 1: AI-диагностика по фото

### Проблема, которую решает
"Я не понимаю, что с моими волосами. Хочу знать, насколько всё плохо."

### Как работает
1. Пользователь загружает фото волос
2. Claude API анализирует (vision + system prompt с экспертизой Елены)
3. Результат: степень повреждения + тип + рекомендации

### System prompt — на основе знаний Елены
Елена описывает 20–30 типовых кейсов с фото. Это становится базой prompt'а. AI оценивает по критериям Елены, а не "из общих знаний".

### Техническая реализация
```
lib/ai/
  ├── diagnosis.ts       # Claude API call + photo handling
  ├── prompts.ts         # System prompts
  └── types.ts           # DiagnosisResult

app/api/ai/diagnose/route.ts  # POST: image → result
components/ai/PhotoUpload.tsx  # Upload + preview
components/ai/DiagnosisResult.tsx  # Визуализация результата
```

## Направление 2: Подбор средств

### Проблема
"Я не знаю, какие средства купить. Боюсь потратить деньги на неподходящее."

### Как работает
1. На основе результатов квиза + AI-диагностики
2. База продуктов с аннотациями Елены (для кого, когда, почему)
3. Персональная подборка из 3–5 средств

### Можно начать без AI
Rule-based подбор: тип повреждения → фильтр по каталогу. Claude API — для более нюансированных рекомендаций.

### Монетизация
Партнёрские ссылки на средства. Елена рекомендует только то, чему доверяет.

### Когда: Phase 3

## Направление 3: Курс для мастеров (B2B)

### Проблема
Мастера хотят углубить экспертизу в восстановлении, но нет системного обучения.

### Продукт
- Профессиональный курс: составы, протоколы, диагностика
- Другая цена (199–299€)
- Другой контент, другой тон
- Возможно: сертификация

### Архитектурная готовность
`Purchase.productSlug` уже поддерживает несколько продуктов. Нужно:
- Новый набор уроков в data.ts
- Логика доступа по productSlug
- Отдельная landing page

### Когда: Phase 3

### Проблема
"Мне нужен индивидуальный подход. Хочу поговорить с мастером."

### Формат
- Онлайн-консультация (Zoom/Google Meet), 30 мин
- Booking через сайт (Calendly embed или кастомный)

Бесплатный контент → Квиз → Курс (49€) → **Консультация (50–100€)** → Запись в студию

## Принципы расширения

### 1. Модульность
Каждая новая фича — отдельный модуль. Не вплетается в существующий код.
```
lib/ai/        ← не трогает lib/lessons/
lib/products/  ← не трогает lib/quiz/
lib/email/     ← не трогает lib/stripe/
```

### 2. Feature flags
```typescript
const FEATURES = {
  aiDiagnosis: false,
  productRecommendations: false,
  emailReminders: false,
  proCourse: false,
}
```
Включил — работает. Выключил — как было.

### 3. Backward compatibility
Новые поля в моделях — опциональные. Старые данные не ломаются.

### 4. Контент отделён от кода
data.ts можно заменить на CMS без изменения компонентов. Интерфейс тот же.

## Productization — как думать

### От "проект" к "продукт"

| Мышление проекта | Мышление продукта |
|-----------------|------------------|
| "Сделать и сдать" | "Запустить и развивать" |
| "Клиент доволен" | "Пользователь получает результат" |
| "Красивый сайт" | "Рабочая система, которая решает проблему" |
| "Готово" | "MVP → данные → улучшение → рост" |

### Паттерн, который можно повторять

Этот же подход работает для:
- Другого beauty-эксперта (skincare, nails, makeup)
- Другой индустрии (fitness, nutrition, education)
- Другого формата (coaching, community, marketplace)

Ядро всегда одно: **эксперт + система + guided digital experience + AI-усиление**.
